# Workflow: Layered Structural Review (結構變更的二級審計)

當修改圖表指令涉及結構性操作（如刪除、搬移、替換、重新排版）時，除了全圖的視覺檢查，還必須單獨審查以下四個核心副作用類別。

## 審計項目核對清單

### 1. 重複內容審查 (Content Duplication)
- **觸發情境**：將元素從位置 A 搬移至位置 B。
- **易犯錯誤**：在位置 B 產生了新元素，但忘記刪除位置 A 的舊元素。
- **驗證方式**：在渲染圖中定位 A 點，確認舊元素已被徹底清除或覆蓋。

### 2. 殘留空白審查 (Leftover Blank Space)
- **觸發情境**：移除或刪除某個面板/子圖/箭頭。
- **易犯錯誤**：元素消失後，留下突兀的大片背景空白或空框架，使畫面顯得支離破碎。
- **驗證方式**：在元素被刪除後，檢查是否需要調整其餘子圖的間距，或將剩餘元素 reflow 以填補空隙。

### 3. 新增元素尺度審查 (Element Scaling)
- **觸發情境**：在現有圖表中插入新區塊、文字框或新 icon。
- **易犯錯誤**：插入的新元素相比於既有元素顯得過大或過小，導致比例失調。
- **驗證方式**：將新元素的 bounding box 與周圍同類型元素進行幾何長寬比對，確保其尺寸比例相差不超過 20%。

### 4. 樣式一致性審查 (Style Mismatch)
- **觸發情境**：新增或置換圖表中的箭頭、線條、邊框或標籤。
- **易犯錯誤**：新箭頭的筆觸粗細、顏色（例如使用了鮮艷的紅藍色，而原圖是莫蘭迪色系）、或字型與原圖風格不搭，使其讀起來像「後期畫蛇添足的標註」。
- **驗證方式**：
  - 提取周圍元素的 CSS 屬性（`stroke`, `stroke-width`, `fill`, `font-family`）。
  - 將相同的 CSS 類別或 style 屬性應用到新元素上。
